📉 **5 UTJECAJA NEPLANIRANIH PROMJENA SCOPE-A:**

*[Prikaži slajd 5]*

**1. 📅 SCHEDULE DELAYS (Kašnjenje rasporeda)**

> Dodatni posao = produženi rokovi

*IT Primjer:*
'Dodavanje Payment modula zahtijeva 3 tjedna dodatnog developmenta. Launch date se pomiče s 1. kolovoza na 22. kolovoza.'

**2. 💰 INCREASED COSTS (Povećani troškovi)**

> Više posla = više resursa = više novca

*IT Primjer:*
'Payment modul zahtijeva:
- 80 dodatnih dev sati × 50€ = 4.000€
- Payment gateway integracija: 500€/mjesečno
- Security audit: 2.000€
UKUPNO: 6.500€+ dodatnih troškova'

**3. 👥 RESOURCE STRAIN (Preopterećenost resursa)**

> Isti ljudi, više posla = burnout, pad produktivnosti

*IT Primjer:*
'Backend developer koji je trebao raditi samo na Grades modulu sada radi na Grades + Payment. Radi 60 sati tjedno. Nakon 3 tjedna, produktivnost pada 40%.'

**4. ⚠️ QUALITY RISKS (Rizici kvalitete)**

> Žurba = greške = bugovi = nezadovoljan klijent

*IT Primjer:*
'Da bi stigli novi deadline, tim preskače code review i unit testing. Rezultat: 47 bug-ova u produkciji, od čega 3 kritična security issue-a.'

**5. 😤 STAKEHOLDER CONFLICTS (Konflikti među dionicima)**

> Različita očekivanja = frustracija

*IT Primjer:*
'PM misli da je Payment modul 'dodatak'. Klijent misli da je 'uključen u original deal'. Sponzor pita zašto projekt kasni. Svi su frustrirani.'

*[Prikaži slajd 5]*

Ima li pitanja o scope change management-u?"

📚 **FACILITATOR NOTES:**

**Za tvoje razumijevanje:**
Scope creep je jedan od TOP 3 razloga zašto IT projekti propadaju. Studentice moraju razumjeti da je 'malo više' uvijek mnogo više nego što izgleda.

**Analogija za razumijevanje:**
"Scope creep je kao kad idete u dućan po kruh i vratite se s punom košaricom. Svaka stvar pojedinačno izgleda mala, ali zbroj je velik."

**Česta pitanja:**

Q: "Što ako je klijent jako važan i ne možemo reći ne?"
A: "Nikad ne kažemo 'ne' direktno. Kažemo: 'Da, to je moguće. Utjecaj na schedule je +3 tjedna i +6.500€. Je li to prihvatljivo?' Odluka je na klijentu."

Q: "Tko je u CCB (Change Control Board)?"
A: "Tipično: PM, Sponsor, Key Stakeholders, Technical Lead. Za male projekte može biti samo PM + Sponsor."

Q: "Što ako promjena dođe usred sprinta?"
A: "U Agile: ide u backlog za sljedeći sprint. Nikad ne dodajemo u trenutni sprint bez izbacivanja nečeg drugog."

**Troubleshooting:**
- **Studentice misle da je proces prekompliciran:** "Za male projekte, koraci 3-4 mogu biti brzi email. Proces se skalira prema veličini projekta."
- **Pitaju o 'quick wins':** "Čak i mali 'quick win' mora biti dokumentiran. Inače, 10 'quick wins' = 1 veliki scope creep."

📊 **5 TEHNIKA ZA PRAĆENJE RASPOREDA:**

*[Prikaži slajd 11]*

**1. 📊 GANTT CHARTS**

> Vizualna vremenska linija koja pokazuje taskove, trajanja i dependencies.

*Što prati:*
- Koji taskovi su završeni (popunjene trake)
- Koji kasne (crvene trake)
- Gdje smo danas vs. plan (vertical 'today' line)

*IT Primjer:*
```
Task            | Tjedan 1-2 | Tjedan 3-4 | Tjedan 5-6 |
----------------|------------|------------|------------|
UI Design       | ██████████ | ▓▓▓▓       |            |
Backend Dev     |            | ██████████ | ██████████ |
Testing         |            |            | ████       |
                      ↑
                TODAY (Week 4) - UI kasni!
```

---

**2. 🛤️ CRITICAL PATH METHOD (CPM)**

> Identificira NAJDUŽI niz ovisnih taskova = minimalno trajanje projekta

*Ključna ideja:*
- Tasks na Critical Path imaju **0 float** (slack)
- Ako task na Critical Path kasni → CIJELI PROJEKT kasni!
- Tasks VAN Critical Path imaju fleksibilnost

*IT Primjer:*
```
Critical Path: Requirements → Design → Backend → Testing → Launch
               (10 dana)     (15 dana) (30 dana) (10 dana) (5 dana)
               = 70 dana MINIMUM

NON-Critical: Documentation (ima 10 dana float-a)
```

---

**3. 📈 EARNED VALUE MANAGEMENT (EVM)**

> Mjeri schedule performance koristeći formule

*Ključne metrike:*
- **SPI (Schedule Performance Index)** = EV / PV
  - SPI > 1 = ahead of schedule ✅
  - SPI < 1 = behind schedule ❌
  - SPI = 1 = on schedule ✓

*Primjer:*
'SPI = 0.8 znači da za svaki 1€ planiranog posla, napravili smo samo 0.80€ posla. Kasnimo 20%!'

---

**4. 🏁 MILESTONE TRACKING**

> Prati ključne faze projekta (checkpoints)

*IT Primjer - EduTrack Milestones:*
| Milestone | Planirani datum | Stvarni datum | Status |
|-----------|-----------------|---------------|--------|
| M1: Design Complete | 15. travnja | 15. travnja | ✅ On time |
| M2: Backend Complete | 30. svibnja | 10. lipnja | ❌ 11 dana kašnjenja |
| M3: Beta Release | 15. srpnja | ? | ⏳ Pending |

---

**5. 🔄 AGILE METHODOLOGIES (Scrum/Kanban)**

🛤️ **CRITICAL PATH METHOD - DETALJNIJE:**

*[Prikaži slajdove 13-16]*

CPM je KLJUČNA tehnika za schedule tracking. Razumijmo je dublje.

**Ključni elementi CPM-a:**

| Element | Opis | Primjer |
|---------|------|---------|
| **Activities** | Taskovi koji se moraju napraviti | Design, Development, Testing |
| **Dependencies** | Veze između taskova | Testing NE MOŽE početi dok Development nije gotov |
| **Durations** | Koliko traje svaki task | Design = 10 dana |
| **ES (Earliest Start)** | Najranije kada task može početi | Design ES = Dan 1 |
| **EF (Earliest Finish)** | Najranije kada task može završiti | Design EF = Dan 10 |
| **LS (Latest Start)** | Najkasnije kada task može početi bez kašnjenja | Design LS = Dan 1 |
| **LF (Latest Finish)** | Najkasnije kada task može završiti bez kašnjenja | Design LF = Dan 10 |
| **Float (Slack)** | Koliko task može kasniti bez utjecaja | Documentation Float = 5 dana |

**Koraci za primjenu CPM-a:**

1. Nabroji sve taskove i trajanja
2. Identificiraj dependencies
3. Nacrtaj network diagram
4. Izračunaj Earliest Start/Finish (Forward Pass)
5. Izračunaj Latest Start/Finish (Backward Pass)
6. Izračunaj Float za svaki task
7. Identificiraj Critical Path (tasks s Float = 0)

| Task | Opis | Trajanje (dana) | Ovisi o |

| A | Requirements Gathering | 5 | - |
| B | UI/UX Design | 10 | A |
| C | Backend Development | 20 | A |
| D | Frontend Development | 15 | B, C |
| E | API Integration | 5 | C |
| F | Testing | 10 | D, E |
| G | Deployment | 3 | F |

```
        ┌──→ B (10) ──┐
        │             ↓
A (5) ──┤             D (15) ──→ F (10) ──→ G (3)
        │             ↑
        └──→ C (20) ──┴──→ E (5) ──┘
```

**Izračun Critical Path:**

*Put 1:* A → B → D → F → G = 5 + 10 + 15 + 10 + 3 = **43 dana**
*Put 2:* A → C → D → F → G = 5 + 20 + 15 + 10 + 3 = **53 dana** ← CRITICAL PATH
*Put 3:* A → C → E → F → G = 5 + 20 + 5 + 10 + 3 = **43 dana**

**Critical Path = A → C → D → F → G = 53 dana (minimum trajanje projekta)**

💡 **Što to znači:**
- Taskovi A, C, D, F, G moraju se pratiti VRLO PAŽLJIVO

Ima li pitanja o CPM-u?"

# ⏰ **11:10-11:35 (25 min) - BUDGET TRACKING - EVM FUNDAMENTALS**

### **Slajdovi 17-23: EVM Techniques for Budget Tracking**

**Notes iz PDF-a:**
_"Earned Value Management (EVM) is a project management technique used to assess a project's performance by comparing the planned progress to the actual progress."_

## **Što reći studenticama (25 min):**

"Treća tema danas: **Kako znamo jesmo li u budgetu?**

Odgovor: **Earned Value Management (EVM)** - najmoćnija tehnika za praćenje projekta!

---

💰 **ŠTO JE EVM?**

*[Prikaži slajd 20]*

**Definicija:**
> EVM je tehnika koja uspoređuje PLANIRANI napredak sa STVARNIM napretkom, mjereći scope, schedule i cost istovremeno.

**Zašto je moćan:**
- Tradicionalno praćenje: 'Potrošili smo 50.000€' → Je li to dobro ili loše? Ne znamo!
- EVM praćenje: 'Potrošili smo 50.000€ za posao vrijedan 45.000€' → Prekomjerna potrošnja!

---

📊 **3 KLJUČNE EVM VRIJEDNOSTI:**

*[Prikaži slajd 21]*

*[Nacrtaj na whiteboard]*

```
     €
     │
     │    PV (Plan) ────────────────
     │                  ┌─── EV (Earned)
     │                 /
     │    AC (Actual) /
     │               /
     │──────────────────────────→ Vrijeme
                    ↑
                  DANAS
```

**1. 📈 PV - PLANNED VALUE (Planirana vrijednost)**

> Budgetirani trošak posla koji je PLANIRAN za određeni datum

*IT Primjer:*
'Do kraja 3. mjeseca planirali smo završiti posao vrijedan 300.000 lei.'

**2. 📊 EV - EARNED VALUE (Zarađena vrijednost)**

> Budgetirani trošak posla koji je STVARNO ZAVRŠEN do određenog datuma

*IT Primjer:*
'Do kraja 3. mjeseca stvarno smo završili posao vrijedan 270.000 lei.'

**3. 💸 AC - ACTUAL COST (Stvarni trošak)**

> Stvarni trošak utrošen za posao završen do određenog datuma

*IT Primjer:*
'Do kraja 3. mjeseca potrošili smo 280.000 lei.'

---

📉 **EVM METRIKE - VARIJANCE:**

*[Prikaži slajd 21-22]*

**COST VARIANCE (CV) - Varijanca troškova**
```
CV = EV - AC
```

| CV vrijednost | Značenje |
|---------------|----------|
| CV > 0 | Under budget ✅ (potrošili manje nego vrijednost završenog posla) |
| CV < 0 | Over budget ❌ (potrošili više nego vrijednost završenog posla) |
| CV = 0 | On budget ✓ |

*Primjer:*
CV = 270.000 - 280.000 = **-10.000 lei** → Over budget za 10.000!

**SCHEDULE VARIANCE (SV) - Varijanca rasporeda**
```
SV = EV - PV
```

| SV vrijednost | Značenje |
|---------------|----------|
| SV > 0 | Ahead of schedule ✅ (napravili više nego planirano) |
| SV < 0 | Behind schedule ❌ (napravili manje nego planirano) |
| SV = 0 | On schedule ✓ |

*Primjer:*
SV = 270.000 - 300.000 = **-30.000 lei** → Behind schedule za 30.000!

---

📊 **EVM METRIKE - INDEKSI:**

**COST PERFORMANCE INDEX (CPI) - Indeks troškova**
```
CPI = EV / AC
```

| CPI vrijednost | Značenje |
|----------------|----------|
| CPI > 1 | Efficient ✅ (dobivamo više vrijednosti po € utrošenom) |
| CPI < 1 | Inefficient ❌ (dobivamo manje vrijednosti po € utrošenom) |
| CPI = 1 | On target ✓ |

*Primjer:*
CPI = 270.000 / 280.000 = **0.964** → Za svaki 1€ potrošen, dobivamo 0.96€ vrijednosti. Inefficient!

**SCHEDULE PERFORMANCE INDEX (SPI) - Indeks rasporeda**
```
SPI = EV / PV
```

| SPI vrijednost | Značenje |
|----------------|----------|
| SPI > 1 | Ahead ✅ (radimo brže nego planirano) |
| SPI < 1 | Behind ❌ (radimo sporije nego planirano) |
| SPI = 1 | On track ✓ |

*Primjer:*
SPI = 270.000 / 300.000 = **0.9** → Radimo na 90% planiranog tempa. Kasnimo!

---

🔮 **EVM FORECASTING - PROGNOZIRANJE:**

*[Prikaži slajd 23]*

EVM nam omogućuje PROGNOZIRATI buduće troškove!

**EAC - ESTIMATE AT COMPLETION**
> Procijenjena ukupna cijena projekta na temelju trenutnog performansa

```
EAC = BAC / CPI
```

Gdje je BAC = Budget At Completion (originalni ukupni budget)

*Primjer:*
BAC = 1.000.000 lei, CPI = 0.964
EAC = 1.000.000 / 0.964 = **1.037.171 lei**

💡 Projekt će koštati ~37.000 lei VIŠE nego planirano!

**ETC - ESTIMATE TO COMPLETE**
> Koliko još trebamo potrošiti da završimo projekt

```
ETC = EAC - AC
```

*Primjer:*
EAC = 1.037.171, AC = 280.000
ETC = 1.037.171 - 280.000 = **757.171 lei** još trebamo

---

📋 **EVM CHEAT SHEET:**

| Metrika | Formula | Interpretacija |
|---------|---------|----------------|
| **CV** | EV - AC | + = under budget, - = over budget |
| **SV** | EV - PV | + = ahead, - = behind |
| **CPI** | EV / AC | >1 = efficient, <1 = inefficient |
| **SPI** | EV / PV | >1 = ahead, <1 = behind |
| **EAC** | BAC / CPI | Projected total cost |
| **ETC** | EAC - AC | Remaining budget needed |

💡 **Zlatno pravilo:**
> Ako je EV u formuli, to je DOBRA stvar (više EV = bolje).
> Ako je AC u formuli kao oduzimanje/dijeljenje, više AC = lošije.

Ima li pitanja o EVM-u prije nego krenemo na vježbu?"

📚 **FACILITATOR NOTES:**

**Za tvoje razumijevanje:**
EVM izgleda komplicirano, ali svodi se na:
- PV = Koliko PLANIRANO završiti do sada
- EV = Koliko STVARNO završili do sada
- AC = Koliko STVARNO potrošili do sada

Usporedba ova 3 broja daje cijelu sliku!

**Analogija:**
"Zamislite da renovirate kuću. Plan je bio da do sada završite kuhinju i kupaonicu (PV=20.000€). Stvarno ste završili kuhinju (EV=12.000€). Potrošili ste 15.000€ (AC). EVM vam kaže: kasnite (EV<PV) i prekomjerno trošite (EV<AC)."

**Česta pitanja:**

Q: "Kako znamo EV?"
A: "EV = % completion × BAC za taj task. Ako je backend 50% gotov i planiran za 20.000€, EV = 10.000€."

Q: "Zašto 'Earned Value' a ne 'Actual Value'?"
A: "Jer mjeri vrijednost POSLA, ne novca. Napravili ste posao 'vrijedan' 10.000€ - bez obzira koliko ste zapravo potrošili."

Q: "Tko koristi EVM u praksi?"
A: "Velike korporacije, government projekti, PMO odjeli. Za male projekte se pojednostavljuje, ali koncept ostaje isti."

**Troubleshooting:**
- **Zbunjene formulama:** "Fokusirajte se na interpretaciju, ne na formule. CPI<1 = loše. SPI<1 = loše. To je dovoljno za početak!"
- **Pitaju o negativnim vrijednostima:** "CV i SV mogu biti negativni. CPI i SPI su uvijek pozitivni (omjeri)."

**Planirano trajanje:** 12 mjeseci

| Metrika | Vrijednost |
|---------|------------|
| **Planned Value (PV)** | 250.000€ (trebali smo napraviti polovicu) |
| **Earned Value (EV)** | 200.000€ (stvarno smo napravili) |
| **Actual Cost (AC)** | 225.000€ (stvarno smo potrošili) |

**Vaš zadatak - izračunajte:**
1. Cost Variance (CV)
2. Schedule Variance (SV)
3. Cost Performance Index (CPI)
4. Schedule Performance Index (SPI)

**1. Cost Variance (CV)**
```
CV = EV - AC
CV = 200.000 - 225.000
CV = -25.000€
```
**Interpretacija:** Projekt je **over budget za 25.000€** ❌

**2. Schedule Variance (SV)**
```
SV = EV - PV
SV = 200.000 - 250.000
SV = -50.000€
```
**Interpretacija:** Projekt je **behind schedule za 50.000€ vrijednosti posla** ❌

**3. Cost Performance Index (CPI)**
```
CPI = EV / AC
CPI = 200.000 / 225.000
CPI = 0.889
```
**Interpretacija:** Za svaki 1€ potrošen, dobivamo samo **0.89€ vrijednosti**. Inefficient! ❌

**4. Schedule Performance Index (SPI)**
```
SPI = EV / PV
SPI = 200.000 / 250.000
SPI = 0.8
```
**Interpretacija:** Radimo na **80% planiranog tempa**. Značajno kašnjenje! ❌

**5. Estimate At Completion (EAC)**
```
EAC = BAC / CPI
EAC = 500.000 / 0.889
EAC = 562.429€
```
**Interpretacija:** Projekt će završiti za **~562.000€** (62.000€ preko budgeta!) ⚠️

**6. Estimate To Complete (ETC)**
```
ETC = EAC - AC
ETC = 562.429 - 225.000
ETC = 337.429€
```
**Interpretacija:** Trebamo još **~337.000€** da završimo projekt

**7. Ukupna ocjena:**

| Metrika | Vrijednost | Status |
|---------|------------|--------|
| CV | -25.000€ | ❌ Over budget |
| SV | -50.000€ | ❌ Behind schedule |
| CPI | 0.889 | ❌ Inefficient (<1) |
| SPI | 0.80 | ❌ Behind (<1) |
| EAC | 562.429€ | ⚠️ +62.000€ over original |

**Zaključak:** Projekt je u LOŠEM stanju! Kasni 20% i troši 11% više nego bi trebao. Ako nastavimo ovim tempom, završit ćemo 62.000€ preko budgeta.

**Preporučene akcije:**
- Provesti Root Cause Analysis - zašto kasnimo?

*Prođi kroz rješenje korak po korak.*

**Pitanja:**

- "Što biste rekli sponzoru projekta?" (Projekt kasni i premašuje budget. Trebamo hitno intervenirati.)
- "Kada biste trebali reagirati - na mjesec 6 ili prije?" (PRIJE! EVM treba pratiti svaki mjesec/sprint)

📚 **FACILITATOR NOTES:**

**Kako voditi aktivnost:**
- Obilazi i provjeri izračune
- Ako griješe s formulama: "Podsjetnik: CV = EV minus AC. EV je uvijek na prvom mjestu."

**Česti problemi:**
- **Krivi redoslijed oduzimanja:** "CV = EV - AC, ne AC - EV!"

**Ključna poruka:**

# 🔄 **BACKUP ACTIVITIES**

## **Ako ostane vremena (5-10 min):**

### **EVM Scenario Game**
"Dam vam EVM podatke, vi kažete je li projekt zdrav ili ne!"

**Scenario 1:** CPI = 1.1, SPI = 0.95
*Odgovor: Under budget ali blago kasni - OK ali pazi na schedule*

**Scenario 2:** CPI = 0.7, SPI = 1.2
*Odgovor: Brži od plana ali OZBILJNO over budget - Problem!*

**Scenario 3:** CPI = 0.9, SPI = 0.9
*Odgovor: Blago over budget I kasni - Treba pažnja*

### **Scope Creep Red Flags Discussion**
"Koje su 'crvene zastave' da se događa scope creep?"

*Moguci odgovori:*
- "Samo još jedna mala stvar..."
- "To neće trajati dugo..."
- "Zar to nije bilo uključeno?"
- Features se dodaju bez dokumentacije
- Tim radi stvari koje nisu u WBS-u

## **Ako grupa napreduje sporije:**

### **Pojednostavi CPM aktivnost:**
- Umjesto da računaju sve puteve, daj im gotov dijagram
- Neka samo identificiraju Critical Path vizualno
- Fokus na KONCEPT, ne na matematiku

- Daj im već izračunate CV i SV
- Neka samo interpretiraju (je li + ili -, što to znači)
- Fokus na "CPI/SPI > 1 = dobro, < 1 = loše"
